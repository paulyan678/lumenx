剧本格式与行业标准

术语英文解释
FountainFountain Markup一种纯文本剧本格式标准（2012年）。用简单的文本规则（如 INT. 表示内景）来标记剧本元素，任何文本编辑器都能写剧本，类似 Markdown 之于文档。
FDXFinal Draft XMLFinal Draft 软件的专有 XML 格式，是好莱坞行业事实标准。制片公司/编剧经纪人通常要求提交 .fdx 文件，支持 FDX 导入导出 = 能接入行业工作流。
场景头Scene Heading / Slug Line剧本中标识场景位置和时间的行，如「INT. 咖啡馆 - 夜」或「内景. 咖啡馆 - 夜」。INT=内景，EXT=外景。
Beat SheetBeat Sheet剧本节拍表。将故事拆解为关键情节节点（beats）的结构化工具，常用 Save the Cat / 三幕式结构等框架。
Script LockingScript Locking剧本锁定。进入制片阶段后“冻结”页码，后续修改用彩色修订页标记，确保各部门参照的页码不变。
WGAWriters Guild of America美国编剧工会。剧本行业工会组织，制定 AI 使用政策、保护编剧权益，其 2023 MBA 协议明确限制了 AI 在剧本创作中的角色。

协作与数据同步技术

术语英文解释
CRDTConflict-free Replicated Data Type无冲突复制数据类型。一种数据结构，允许多人同时编辑同一文档而不产生冲突，是 Google Docs / Figma / Notion 实时协作的核心技术。
YjsYjs一个开源的 CRDT 实现库（JavaScript），提供实时协作编辑能力。Tiptap 编辑器内置支持 Yjs，可让多人像在 Google Docs 里一样同时编辑剧本。
LoroLoro CRDT新一代高性能 CRDT 引擎（Rust 编写），Laper 采用。与 Yjs 目标相同（实时协作），但宣称在结构化文档场景有更好的性能表现。运行于 Web Worker 中不阻塞主线程。
AI as CRDT Peer—让 AI 作为文档的"协作者节点"加入编辑，像人类协作者一样参与文档修改。这是 2026 年新兴架构方向。

编辑器技术

术语英文解释
TiptapTiptap Editor基于 ProseMirror 的开源富文本编辑框架。类比：Notion 块编辑器的底层引擎，支持自定义 Block 类型、嵌套节点、实时协作。
ProseMirrorProseMirrorTiptap 的底层引擎。一个极度可定制的文档编辑工具包，灵活度极高但上手门槛也高。
Block-BasedBlock-based Editing块编辑范式。文档由独立的“块”组成（段落/图片/代码等），每块可独立拖拽、转换类型。Notion/Craft 是典型代表。
ASTAbstract Syntax Tree抽象语法树。将文本/代码解析为结构化的树形数据，Fountain.js 输出的 tokens 就是一种简化 AST。

其他缩写与概念

术语英文解释
CJKChinese-Japanese-Korean中日韩文字系统的统称。在剧本软件中指对这些语言的排版、字符宽度、分行规则等特殊支持。
CAGRCompound Annual Growth Rate复合年增长率。衡量市场多年平均增速的指标，17–19% 表示每年市场规模增长这个比例。
PLGProduct-Led Growth产品驱动增长。用户先免费使用产品，体验价值后再付费，与“无需注册即可试用”策略相关。
Multi-agentMulti-agent AI多智能体架构。多个 AI 代理各司其职（格式化、续写、评估等），协同完成复杂任务，而非单一 LLM 处理所有事情。
AI-NativeAI-Native从设计之初就以 AI 为核心构建（而非后期加装 AI 功能）。整个产品体验围绕 AI 能力设计。
inkink (inkle)一种叙事脚本语言，用统一语法表达叙事内容和逻辑。被评为 AI 叙事集成友好度最高的工具，可导出 JSON 与任何引擎对接。

现状评估

ScriptProcessor.tsx 仍为纯 textarea，但已具备完整视频管线和两阶段实体提取。

已有优势

  • 两阶段实体提取（extractPreview → confirm）
  • 系列感知（ReconcileModal 跨集合并）
  • 完整视频管线（剧本→分镜→视频）
  • 两层保存（Zustand + blur 持久化）
  • 完整 REST API（可编程集成）

关键缺口

  • 格式引擎：无自动排版（P0）
  • 语法高亮：元素无区分（P0）
  • Fountain 解析：无结构识别（P0）
  • 格式导出：无 FDX/PDF（P1）
  • 内联 AI：无上下文续写（P1）
  • 协作：无实时多人（P3）

三层解析架构

第一层
规则优先
Fountain.js 语法规则确定性解析，零延迟、零成本，覆盖 80% 常见情形。
第二层
LLM 兆底
规则无法确定时（非标准格式、混合语言）调用 LLM 推断元素类型。
第三层
用户做主
最终分类决定权归用户，UI 提供快速修正快捷键。
设计原则
可预测 > 智能
确定性规则永远优先于 AI 推断，避免「不可控」体验。

设计约束

应做

  • 内容（剧本文本）始终占据视觉主体
  • 渐进式披露：默认极简，高级设置可展开
  • 每个操作有即时视觉反馈（光标/动画/进度）
  • 与视频管线无缝衔接（剧本→分镜一键流转）

避免

  • 模仿传统届面工具的多面板布局
  • AI 功能强制弹出/主动推送（尊重主权）
  • 过度动画分散写作注意力
  • 与视频预览完全分离的孤立编辑体验
